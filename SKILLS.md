## Project-Specific Patterns

### Navigation Pattern (Hardcoded Whitelist)
The Navbar intentionally ignores Firestore settings to prevent flickering. Changes to navigation must be made in `src/components/Navbar.jsx` by modifying the `ALLOWED_LINKS` array.

### Version Footer Pattern
Footer uses `useMemo` to generate a unique timestamp on each page load:
```javascript
const VERSION = useMemo(() => {
    const now = new Date();
    // Format: YYYYMMDD-HHMMSS
}, []);
```

### Gallery Category Pattern
Categories are hardcoded (8 categories), but items are fetched from Firestore at runtime using `getDoc` from the `gallery_categories` collection.

### Admin Access Pattern
Routes like `/admin` and `/admin/media` are protected server-side by checking `user.role === 'owner'` in the component.

## Common Tasks & Solutions

| Task | Solution |
|------|----------|
| Add new nav link | Edit `ALLOWED_LINKS` in Navbar.jsx |
| Add new gallery category | Edit `GALLERY_CATEGORIES` in Gallery.jsx + Firestore |
| Modify pricing | Edit Pricing.jsx or Firestore `config/credits` |
| Upload media | Use AdminMedia page (owner only) |
| Manage categories | Use AdminMedia > Categories tab |
| Add new page | Add route in App.jsx + create page component |
